Navigation Pattern (Mobile)

Use navigate(url, exitClass, entryClass) from shared.js for all page transitions. Transition classes animate the .phone element.

Direction exitClass entryCls Usage
Forward → exit-left enter-right Push deeper into a flow
Back ← exit-right enter-left Back/cancel buttons
Cross-fade exit-fade enter-fade Tab switches, avatar tap
navigate('02-wallet.html', 'exit-fade', 'enter-fade')

Mobile Component Reference

Key layout primitives for the 375×812px phone frame. All rendered inside .phone.

.status-bar
44px tall, fixed to top of .phone. Contains .status-time and .status-icons. Copy the SVG icons verbatim — do not modify.
.navbar
44px tall. Flexbox row. Contains .avatar or .icon-btn elements. Add position:relative to center a title with .navbar-title + absolute positioning.
.balance-section
Optional. Shows .balance-label and .balance-amount. The .balance-currency span is clickable for currency switching.
.scroll-area
flex: 1; overflow-y: auto — the main scrollable content region. Use padding: 0 20px for edge insets.
.bottom-nav
5 tabs. Active tab shows .nav-tab-label + .nav-dot. Inactive tabs show .nav-tab-icon with a 24px image. Use Figma asset URLs with onerror SVG fallback.
.toast
Centered bottom toast. Call showToast('message') to show. Automatically hides after 1.8s. Required on every screen.
